这个博客怎么用？

一句话：你只管往两个文件夹里放文件，网页列表会自己更新。不用写代码，不用懂技术。

🧺 先理解一件事

把这个博客想象成一个文件柜：两个抽屉，一张会自动更新的目录。

📂 你放文件 文章丢进 posts/
网页丢进 projects/
→
⚙️ 机器整理目录 自动扫描文件夹
生成一张清单
→
🌐 网页自动显示 首页、文章页
自动出现你的内容
中间那步「整理目录」是全自动的：本地预览时自动做一次，发布到网上时也自动做一次。你完全不用管。

✋ 你总共只需要碰 3 个地方

除了这三处，其它文件都是机器用的，看不懂也没关系，别动就行。

📝

想发一篇随笔

往 posts/ 丢一个 .md 文件

🎨

想放一个网页/演示

往 projects/ 丢一个文件夹

⚙️

想改博客名/头像

编辑 site.json

📝 手把手：发一篇随笔

  1. 在 posts/ 文件夹里新建一个文本文件，比如 posts/我的第一篇.md。

    （.md 是 Markdown 格式，就是普通文本，用任何编辑器都能写。）

  2. 正常写正文就行。什么都不额外设置也能用 —— 标题会默认取文件名，日期取保存时间，摘要自动抓第一段。
  3. （可选）想让标题、日期更好看？在文件最开头加这样一小段，每一行都能省略： 
    ---
title: 随笔标题
date: 2026-06-20
tags: [随笔, 生活]
summary: 一句话摘要，会显示在首页卡片上。
---

正文从这里开始……
  4. 保存。完成！本地预览或发布后，首页会自动出现这篇文章。
数学公式照常写：行内用 $...$，单独成行用 $$...$$，网页会自动渲染成漂亮的公式。
小技巧：文件名起成 2026-06-20-标题.md 这种带日期前缀的，日期也会被自动识别。

🎨 手把手：放一个网页 / 交互演示

  1. 在 projects/ 里新建一个文件夹，名字随意，比如 projects/我的小工具/。
  2. 把你做好的网页放进去，主文件命名为 index.html。（卡片标题会自动取这个网页的 <title>。）
  3. 完成。它会自动出现在首页的「作品 / 演示」区，发布后还能通过网址直接打开。

🔗 文章和网页，可以配对也可以各自独立

系统会自动判断，你不用做任何额外操作：

🤝
配对：文章 + 同名网页
posts/某文.md 和 projects/某文/ 取同样的名字 → 自动关联，文章页会出现「打开演示」按钮。
📄
只有文章
只放 posts/某文.md → 就是一篇纯文字随笔。
🖼️
只有网页
只放 projects/某作品/ → 自动出现在首页「作品 / 演示」区。

👀 怎么在自己电脑上先看看效果

改完东西，想发布前先预览长啥样：

  1. 双击文件夹里的预览脚本：Windows 双击 本地预览.bat，macOS 双击 本地预览.command。它会自动整理目录、开一个本地网页、弹出浏览器。
  2. 看完想关闭：回到弹出的黑色终端窗口，按 Control + C。
为什么不能直接双击 index.html？网页需要读取文件清单，浏览器出于安全限制不允许这样直接读本地文件，必须经过一个「本地小服务器」—— 那个预览脚本（Windows 的 .bat / macOS 的 .command）就是帮你一键启动它。

🚀 怎么把它发布到网上（只需设置一次）

  1. 注册并登录 GitHub，新建一个仓库，名字必须是 你的用户名.github.io。
  2. 把这个文件夹里的所有文件上传到该仓库（放在 main 分支）。
  3. 进入仓库的 Settings → Pages，把 Source（来源）设为 GitHub Actions。
  4. 等一两分钟，打开 https://你的用户名.github.io 就能看到你的博客了。
设置好以后，发新文章 = 丢文件 + 上传。网站会自动整理目录、自动更新，你什么都不用配。

🎀 想做点小定制

改博客名字 / 副标题 / 头像 / 链接：编辑 site.json。头像可以填一个 emoji（如 ✦），也可以填一张图片的网址或路径。

换配色：打开 assets/css/style.css，最上面那几行 --accent 之类的就是全站颜色，改完即生效。不想自己折腾就直接告诉作者帮你调。

🔍 好奇：背后是怎么自动的？

点开看一个简短、无痛的解释

网页放在 GitHub 上是「纯静态」的 —— 好处是免费、稳定、不会坏。但纯静态网页有个天生的限制：它没法自己「看一眼文件夹里有哪些文件」。

所以我们用一个叫 generate.py 的小程序，帮网页「数一遍文件夹、列一张清单」（就是 content.json）。网页只要读这张清单就知道要显示哪些内容。

这个「数文件、列清单」的动作，在你本地预览和上传发布时都会自动跑一次，所以你永远不用亲自去碰那张清单。

❓ 常见问题

我完全不会编程，能用吗？
能。你只需要会「新建文本文件」和「把文件拖进文件夹」。写文章就像写记事本。
写文章必须填那段 title/date 吗？
不必。什么都不填也能正常显示，那段只是让标题、日期更好看的可选项。
我删掉一篇文章会怎样？
把 posts/ 里对应的 .md 删掉即可，清单会自动更新，网页上也就不显示了。
content.json、generate.py 这些是什么，我要管吗？
不用管，那是机器自动用的。你只碰 posts/、projects/、site.json 三处。
公式 / 代码 / 图片能放吗？
公式和代码都支持。图片可以放进 projects/ 或自建文件夹再在文章里引用。

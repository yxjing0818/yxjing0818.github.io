# 我的随笔小馆

一个**纯静态**个人博客：随笔用 Markdown 写（支持 LaTeX 公式），自己做的静态网页/交互演示放进 `projects/` 直接展示。

**你只负责往文件夹里放文件，列表会自动生成** —— 本地预览时自动生成、push 后由 GitHub Action 自动生成。
你永远不用手动维护任何清单文件。

## 你只需要碰这三个地方

| 想做的事 | 操作 |
|---|---|
| 发一篇随笔 | 往 `posts/` 丢一个 `.md` 文件 |
| 放一个静态网页/演示 | 往 `projects/` 丢一个文件夹（里面有 `index.html`） |
| 改博客名字/头像/链接 | 编辑 `site.json` |

其余文件（`generate.py`、`content.json`、`index.html`、`post.html`、Action）都是机器用的，**不用碰**。

## 文章和演示的关系（不必成对）

- `posts/某文.md` + `projects/某文/`（**同名**）→ 自动关联，文章页出现「打开演示」按钮
- 只有 `posts/某文.md` → 纯随笔
- 只有 `projects/某作品/` → 自动出现在首页「作品 / 演示」区

## 写一篇随笔

最简单：往 `posts/` 丢个 `.md`，正常写正文即可（公式 `$...$` 行内、`$$...$$` 独立成段）。
不写任何额外信息也能用：标题默认取文件名，日期默认取文件修改时间，摘要自动取第一段。

想让标题/日期/标签/摘要更好看，就在文件**最开头**加一小段（每行都可省略）：

```markdown
---
title: 随笔标题
date: 2026-06-20
tags: [标签A, 标签B]
summary: 一句话摘要，显示在首页卡片上。
---

正文从这里开始……
```

> 小技巧：把文件名起成 `2026-06-20-标题.md` 这种带日期前缀的，也会自动识别日期。

## 放一个静态网页 / 演示

把做好的网页放成 `projects/某名字/index.html`（卡片标题自动取该页 `<title>`）。
部署后通过 `https://yxjing0818.github.io/projects/某名字/` 访问。
如果它对应某篇随笔，**把文件夹和随笔取同名**就会自动关联。

## 本地预览

- **Windows：双击 `本地预览.bat`**（不需要手动安装或配置 Python）
- **macOS：双击 `本地预览.command`**

它会先自动生成清单，再开服务器并打开浏览器。关闭：在弹出的窗口里按 `Ctrl+C`，或直接关闭窗口。

> Windows 脚本会自动从 python.org 下载项目私有运行时到 `.runtime/`（仅首次需要）。不修改系统环境变量，不需要 `py` 命令，也不依赖 PowerShell。

> 为什么不能直接双击 index.html？页面要用 fetch 读取清单和 `.md`，浏览器禁止 `file://` 直接读，必须经过本地服务器 —— 这正是该脚本做的事。

## 部署到 GitHub Pages（一次性设置）

1. 本项目使用 `yxjing0818/yxjing0818.github.io` 仓库，push 到 `main` 分支即可。
2. 仓库 Settings → Pages → Source 选 **GitHub Actions**。
3. 之后每次 push，Action 会自动扫描文件夹、生成清单并部署。访问 `https://yxjing0818.github.io`。

> 也就是说，发新文章的完整流程 = 丢文件 → push。列表自动更新。

## 换配色

打开 `assets/css/style.css`，最顶部 `:root` 里那组变量（`--accent` 等）就是全站配色，改完即生效。
